# Reporting and dashboards

## Description

As a backoffice team lead or campus administrator, I need dashboards and reports so that workload, pipeline health, service performance, and retention risk can be monitored.

## Acceptance criteria

- The system provides reports for students by journey stage, pipeline volumes, outstanding tasks, open cases by category and age, active risk flags, average case resolution time, status movement trends, and withdrawals or deferments by reason.
- Reports support filtering by campus, program, date range, assigned owner, and status where those fields are relevant.
- Team lead dashboards are limited to the teams, queues, campuses, or programs the team lead is permitted to oversee.
- Campus administrator dashboards can include institution-wide summaries.
- Dashboard totals and report outputs use the same source definitions as operational records and queue views.
- Report results reflect data committed to the system at the time the report is run.
- Each report defines the date field used for filtering so users can distinguish opened date, due date, resolved date, and status effective date.
- Exported report data must honor the same permission rules as on-screen reporting.
