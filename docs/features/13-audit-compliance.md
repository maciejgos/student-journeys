# Audit and Compliance

## Description

As an administrator or compliance stakeholder, I need sensitive actions to be auditable so that operational changes can be reviewed and the system can support campus retention and privacy obligations.

## Acceptance criteria

- The system keeps an audit log for create, update, assignment, status, and permission-sensitive actions.
- Audit entries record who changed what and when.
- The system can support data retention rules defined by campus policy.
- Personally identifiable and sensitive student data is protected according to platform security controls.
