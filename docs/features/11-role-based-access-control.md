# Role-Based Access Control

## Description

As a campus administrator, I need role-based access control so that workers, team leads, and administrators can access only the data and actions appropriate to their responsibilities.

## Acceptance criteria

- The system supports at least Backoffice Campus Worker, Backoffice Team Lead, and Campus Administrator roles.
- Backoffice workers can view and update permitted student records, manage tasks, interactions, and cases, upload permitted documents, and run standard operational reports.
- Team leads inherit worker capabilities and can reassign work, manage escalations, view team dashboards, and override selected workflow transitions.
- Campus administrators have configuration access for users, roles, statuses, queues, categories, templates, audit logs, and system settings.
- Permissions are enforced consistently across UI, API, and exported data.
