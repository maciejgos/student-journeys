# Case and Issue Management

## Description

As a backoffice worker or team lead, I need to manage operational cases on student records so that blockers, risks, and support issues can be tracked through resolution.

## Acceptance criteria

- Users can open a case against a student record.
- Cases support category, priority, assignee, status, and escalation data.
- Users can record case notes and attach supporting documents.
- Cases track SLA target dates and actual resolution dates.
- Users can escalate a case to a team lead or specialized queue.
- The feature supports cases such as missing documents, payment issues, registration blockers, attendance concerns, and support requests.
