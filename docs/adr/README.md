Here we store all architecture decision records made during the application design and implementation process.

Use [TEMPLATE.md](./TEMPLATE.md) for new entries or run `npm run codex:feature:start -- --title "Feature title"` to scaffold a related ADR automatically when starting a feature. The ADR should be kept in sync with the AI-native workflow documented in `docs/ai-native-sdlc-pdlc.md` whenever the work changes a long-lived technical boundary.
