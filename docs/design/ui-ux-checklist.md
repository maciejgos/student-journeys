# UI and UX checklist

Use this checklist when creating or reviewing user-facing changes.

- Is the primary user task obvious from the screen?
- Are the main actions easy to distinguish from secondary actions?
- Are empty, loading, success, validation, and failure states covered?
- Does the layout still work on narrow screens?
- Are labels, helper text, and error messages specific and readable?
- Can a keyboard user reach and understand the key interactions?
- Does the change fit existing product patterns unless a deliberate design change is documented?
- If the feature is new or complex, is there a matching note in `docs/design/` created from `docs/design/TEMPLATE.md`?
