# Design notes

Use this directory for feature-level UI and UX notes when the change is significant enough to benefit from an explicit design pass before implementation.

Start new notes from [TEMPLATE.md](./TEMPLATE.md).

Typical contents:

- user flow notes,
- screen inventory,
- interaction states,
- responsive behavior,
- accessibility expectations,
- visual direction constraints for implementation.

Pair these notes with the AI-native delivery flow in `docs/ai-native-sdlc-pdlc.md`. Design notes are optional for small UI tweaks and expected for larger user-facing changes.
