# Design checklist

Before implementation, confirm:

- the primary user and task are explicit,
- the entry point and destination screen are known,
- the main interaction path is described,
- empty, loading, success, validation, and failure states are covered,
- responsive behavior is defined for narrow screens,
- accessibility expectations are stated for keyboard, labels, and focus behavior,
- copy and visual tone match the existing product unless the feature intentionally changes them.
